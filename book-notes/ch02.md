Directories in the kernel source tree (2.6)

| Directory     | Description                                   |
| ------------- | --------------------------------------------- |
| block         | Block I/O layer                               |
| arch          | Architecture-specific source                  |
| crypto        | Crypto API                                    |
| Documentation | Kernel source documentation                   |
| drivers       | Device drivers                                |
| firmware      | Device firmware needed to use certain drivers |
| fs            | The VFS and the individual filesystems        |
| include       | Kernel headers                                |
| init          | Kernel boot and initialization                |
| ipc           | Interprocess communication code               |
| kernel        | Core subsystems, such as the scheduler        |
| lib           | Helper routines                               |
| mm            | Memory management subsystem and the VM        |
| net           | Networking subsystem                          |
| samples       | Sample, demonstrative code                    |
| scripts       | Scripts used to build the kernel              |
| security      | Linux Security Module                         |
| sound         | Sound subsystem                               |
| usr           | Early user-space code (called initramfs)      |
| tools         | Tools helpful for developing Linux            |
| virt          | Virtualization infrastructure                 |

Additional directories in the 6.18 kernel

| Directory | Description                                                        |
| --------- | ------------------------------------------------------------------ |
| certs     | Supports module signing and verification which didn't exist in 2.6 |
| io_uring  | Implementation of the io_uring async I/O interface                 |
| LICENSES  | Legal metadata                                                     |
| rust      | Support for writing kernel code in rust (mostly drivers)           |

Addition topics that make the kernel unique and different:

- no C standard library - printk() instead of printf()
- inline functions
- inline assembly
- branch annotation
- no floating point arithmetic
- small fixed size stack
